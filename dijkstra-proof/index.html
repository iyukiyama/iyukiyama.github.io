Dijkstra算法正确性证明

发表于2022-05-29|更新于2022-11-13|算法与数据结构
|字数总计:2.2k|阅读时长:8分钟|阅读量:

算法描述

Dijkstra算法(狄杰斯特拉算法)：
一种基于广度优先搜索和贪婪思想的求解 无负边有向赋权图单源最短路径 的算法。
算法将所有顶点区分为「到源点 sss 的最短距离」(以后简称「距离」) 已确定 和 未确定 的顶点。算法开始前所有顶点的距离均未确定(一般置为InfinityInfinityInfinity)，初始时置 sss 的距离为 0。以一个while循环查询当前 是否有距离未确定 的顶点，若有则将其中距离最小者 vvv 选为 当前顶点，并使其距离已知。然后以BFS的方式松弛 vvv 的邻接顶点 www 并更新 www 的前驱为 vvv。当不再有未确定距离的顶点时算法结束，此时每一个顶点的距离均最小。通过递归寻找节点的前驱可以得到 sss 到该顶点的具体的最短路径。

松弛操作 是最短路径算法的关键，在确定当前顶点 vvv (最新成为已确定距离的顶点) 后立即操作，目的是更新 vvv 的邻接顶点 www 的距离和其前驱。如下图，sss 经过若干个顶点到 aaa 和 bbb，aaa 和 bbb 邻接 ccc。假设此时 dw=Inifinitydw = Inifinitydw=Inifinity，da=5da = 5da=5，db=10db = 10db=10。

aaa 先于 bbb 成为当前顶点，由于 da+∣(a,w)∣=15<Infinityda + |(a, w)| = 15 < Infinityda+∣(a,w)∣=15<Infinity，故 aaa 松弛 dwdwdw，并将 www 的前驱置为 aaa，w.pre=aw.pre = aw.pre=a。

bbb 成为当前顶点时，由于 db+∣(b,w)∣=11<15db + |(b, w)| = 11 < 15db+∣(b,w)∣=11<15，故 bbb 松弛 dwdwdw，并将 www 的前驱置为 bbb，w.pre=bw.pre = bw.pre=b。

一个直观的观察是，来自 www 入边的松弛，使得 dwdwdw 不断更新至所有可能的路径距离的最小值。以下严格证明该算法正确性。


算法正确性证明

利用数学归纳法(结合反证法)证明。

本证明参考了这个帖子。

a) 数学归纳法的证明过程

  1. 起始验证。对于命题 P(n)P(n)P(n)，当 n=1n = 1n=1 时命题 PPP 成立。

  2. 假设命题成立。假设命题 P(n)P(n)P(n) 在 n=m(m>1,m∈N)n = m (m > 1, m ∈ N)n=m(m>1,m∈N) 时成立。

  3. 递推证明。根据2的假设，若能证明 n=m+1n = m + 1n=m+1 时命题 PPP 成立，则命题得证。

例如，有命题 P：1+2+3...+n=n∗(n+1)/2P：1+2+3...+n = n*(n+1)/2P：1+2+3...+n=n∗(n+1)/2，按照数学归纳法证明如下：

  1. 起始验证。当 nnn 等于 111 时，1=1∗(1+1)/21 = 1*(1+1)/21=1∗(1+1)/2，命题成立。

  2. 假设命题成立。假设命题等于 mmm 时成立，1+2+3+...+m=m∗(m+1)/21+2+3+...+m = m*(m+1)/21+2+3+...+m=m∗(m+1)/2。

  3. 递推证明。根据2的假设，如果能证明 n=m+1n = m+1n=m+1 时命题正确，则命题 PPP 成立。

    证明：在2所示式子左右两边加上 m+1m+1m+1，得到 1+2+3+...+m+(m+1)=m∗(m+1)/2+(m+1)1+2+3+...+m+(m+1) = m*(m+1)/2 + (m+1)1+2+3+...+m+(m+1)=m∗(m+1)/2+(m+1)

    等号右边可以写成 (m+1)∗(m+2)/2(m+1)*(m+2)/2(m+1)∗(m+2)/2，显然该形式就是将 n=m+1n = m+1n=m+1代入原命题 PPP 的形式，证毕。


b) 证明Dijkstra算法的正确性

命题 PPP：Dijkstra算法第 nnn 次进入while时，会将第 nnn 个顶点加入距离已确定顶点集合 AAA 中，此时对于顶点 ∀v∈A(∀v ∈ A(∀v∈A(共 nnn 个)，总有 dv=δvdv = δvdv=δv。

※ dvdvdv 表示由Dijkstra算法得到的最短距离估计，对于源点 sss ，在程序开始时赋予 ds=0ds = 0ds=0，对于其他顶点，由松弛操作得到。δvδvδv 表示实际的顶点 vvv 到源点的最短距离。

  1. 起始验证。当 nnn 等于 111 时，AAA 集合中只有源点 sss 自身，ds=0ds = 0ds=0 (程序开始时赋值得到)，且知 δv=0δv = 0δv=0，故 n=1n=1n=1 时命题正确。

  2. 假设命题成立。假设命题 PPP 在 nnn 等于 mmm 时，P(m)P(m)P(m) 成立，即算法经过 mmm 次while，得到具有 mmm 个顶点的集合 AAA，对于顶点 ∀v∈A∀v ∈ A∀v∈A (共 mmm 个)，总有 dv=δvdv = δvdv=δv。

  3. P(m+1)P(m+1)P(m+1) 递推证明。根据 2 的假设，如果能证明第 m+1m+1m+1 个顶点 uuu 被放入集合 AAA 时有 du=δudu = δudu=δu，则命题 PPP 成立。

    更详细地，∣A∣=m|A| = m∣A∣=m 时，在点集 B(B=S−A)B (B = S - A)B(B=S−A) 中根据算法规则找到距离最短的顶点 uuu，将该顶点将作为第 m+1m+1m+1 个顶点放入A中，放入后 ∣A∣=m+1|A| = m + 1∣A∣=m+1，如果能证明 du=δudu = δudu=δu，使得 P(m+1)P(m+1)P(m+1) 成立，则对于顶点 ∀v∈A∀v ∈ A∀v∈A (共 m+1m+1m+1 个)，有 dv=δvdv = δvdv=δv。

    以反证法证明之。


    3.1 🤔 假设 m+1m+1m+1 时 du=δudu = δudu=δu 不成立，即有如下式(1)， 之后的目标是 根据已知条件导出某种矛盾情形，推翻该假设。

    (1) δu<duδu < duδu<du

    ※ δuδuδu 是实际的 uuu 到源点的最短距离，du=δudu = δudu=δu 不成立时只能是 δu<duδu < duδu<du。算法保证了从 sss 到 uuu 的过程一定是一条由图中的有向边构成的 连续路径，只要是连续路径，无论有多少条这样的路径，一定有一条最短路径，其长度记作 δuδuδu。并使得其他路径长度必大于等于 δu。δu。δu。


    3.2 🤔 根据3.1的假设，存在一条从源点 sss 到 uuu 的路径 PuPuPu，该路径是 sss 到 uuu 的最短路径，即 ∣Pu∣=δu<du|Pu| = δu < du∣Pu∣=δu<du。路径 PuPuPu 一定有不在 AAA 集内的顶点 (至少有 uuu 不在 AAA 集中，注意此时 uuu 还未被选入 AAA 中)，同时也有在 AAA 集中的点 (至少有 sss 点在 AAA 集中)，可以假设 PuPuPu 经过 xxx 和 yyy，其中 xxx 在A中 (可以是 sss)，yyy 在B中 (可以是 uuu 本身)，yyy 到 uuu 的过程中也可以再进入 AAA，如下图。PxPxPx 为 PuPuPu 在顶点 xxx 结束的子路径，因为路径 Px+(x,y)Px + (x, y)Px+(x,y) 为路径 PuPuPu 的一部分，所以有：

    (2) ∣Px∣+∣(x,y)∣≤∣Pu∣=δu|Px| + |(x, y)| ≤ |Pu| = δu∣Px∣+∣(x,y)∣≤∣Pu∣=δu

    这是显然的，因为 Px+(x,y)Px + (x, y)Px+(x,y) 是 PuPuPu 的一部分，当 y=uy=uy=u 时取到等号。



    3.3 🤔 在 xxx 之前被选中进入 AAA 集内时，对其邻接顶点 yyy 执行过 松弛操作，该操作会比较 dx+∣(x,y)∣dx + |(x, y)|dx+∣(x,y)∣ 是否小于 dydydy，若小于则以 dx+∣(x,y)∣dx + |(x, y)|dx+∣(x,y)∣ 更新 dydydy 的值，所以如果更新了，更新之后有 dy=dx+∣(x,y)∣dy = dx + |(x, y)|dy=dx+∣(x,y)∣ ，如果没更新，说明 dy<dx+∣(x,y)∣dy < dx + |(x, y)|dy<dx+∣(x,y)∣。假设之后 yyy 还会被 yyy 的其他前驱顶点更新 dydydy 值 (当该前驱顶点进入 AAA 集时)，那 dydydy 只会变得更小，所以一定有：

    (3) dy≤dx+∣(x,y)∣dy ≤ dx + |(x, y)|dy≤dx+∣(x,y)∣

    比较式 (2) 和式 (3) 中的 ∣Px∣|Px|∣Px∣ 和 dxdxdx，因为 dx=δxdx = δxdx=δx (由步骤2的 P(m)P(m)P(m) 假设给出，顶点 xxx 是 P(m)P(m)P(m) 假设的 mmm 个顶点之一)，而 PxPxPx 只是若干从 sss 到 xxx 的路径之一，因此必有 d(x)≤∣Px∣d(x) ≤ |Px|d(x)≤∣Px∣，当 PxPxPx 恰是 sss 到 xxx 的最短路径时取到等号。所以根据式 (2) 和式 (3) 有：

    dy≤dx+∣(x,y)∣≤∣Px∣+∣(x,y)∣≤∣Pu∣dy ≤ dx + |(x, y)| ≤ |Px| + |(x, y)| ≤ |Pu|dy≤dx+∣(x,y)∣≤∣Px∣+∣(x,y)∣≤∣Pu∣，即

    (4) dy≤∣Pu∣=δudy ≤ |Pu| = δudy≤∣Pu∣=δu

    3.4 🤔 顶点 yyy 与 uuu 均在 BBB 集中，根据算法规则，uuu 作为第 m+1m+1m+1 个定点被放入 AAA 集中时，其在 BBB 集中相比于BBB 集中的其他顶点(自然也包括 yyy )，到源点 sss 的距离最小，显然有：

    (5) du≤dydu ≤ dydu≤dy

    结合式 (1)，式 (4)，式 (5)得到：

    (6) δu<du≤dy≤∣Pu∣=δuδu < du ≤ dy ≤ |Pu| = δuδu<du≤dy≤∣Pu∣=δu ，即 δu<δuδu < δuδu<δu

至此，由3.1的假设 「d(u)=δ(u)d(u) = δ(u)d(u)=δ(u) 不成立」导出了 矛盾，所以 d(u)=δ(u)d(u) = δ(u)d(u)=δ(u) 是成立的，Dijkstra算法正确性得证。
