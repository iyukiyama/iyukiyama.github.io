树状数组 是一种能够高效求解「区间问题」的数据结构。「区间问题」指的是对于大小为 nnn 的输入数组 numsnumsnums ，通过其上执行「区间求和」、「区间修改」等操作 (通过不同类型的树状数组) 来处理的问题，解决区间问题的过程中通常还伴随着针对单个元素的「单点查询」、「单点修改」这两种单点操作。若直接根据下标操作 numsnumsnums ，则单点操作时间复杂度为 O(1)O(1)O(1) ，而区间操作为 O(n)O(n)O(n) ；若采用「前缀和」，则区间操作为 O(1)O(1)O(1) ，而单点操作为 O(n)O(n)O(n) 。

本文将介绍的树状数组，利用 numsnumsnums 的 下标二进制表示及其位运算 ，十分巧妙地将输入区间划分为 nnn 个子区间，使得这些区间构成一棵或多棵多叉树，这些树通过一个数组表达，即所谓「树状数组」。借助树状数组的逻辑树形结构，能够 同时实现 O(logn)O(logn)O(logn) 时间复杂度的单点操作与区间操作 。

树状数组

树状数组 (二元索引树 / 二元下标树 / Binary Indexed Tree, BIT / Fenwick Tree): 树状数组虽名为数组，但从其英文名 (Binary Indexed Tree) 可看出它本质上是一种被表达为树的数据结构。对于大小为 nnn 的序列 numsnumsnums ，最基本的树状数组以 O(logn)O(logn)O(logn) 时间复杂度同时支持如下两种操作。

  • 更新 numsnumsnums 中单个元素的值，即 单点修改 。
  • 求 numsnumsnums 任意区间的元素值之和，即 区间查询 。

对于这两种操作，最简单的做法是直接根据下标操作 numsnumsnums ，单点修改的时间复杂度为 O(1)O(1)O(1) ，区间查询为 O(n)O(n)O(n) 。此外，我们也可以利用「前缀和」来完成。首先计算出 numsnumsnums 的前缀和数组 preSumpreSumpreSum ，那么求 [l,r][l , r][l,r] 区间和，即是求 preSum[r]−preSum[l−1]preSum[r] - preSum[l-1]preSum[r]−preSum[l−1] ，时间复杂度为 O(1)O(1)O(1) 。但单点修改 nums[i]nums[i]nums[i] 时，需要更新 preSum[i]preSum[i]preSum[i] 及之后的前缀和数组元素 (否则之后求区间和会出错)，这使得单点修改时间复杂度为 O(n)O(n)O(n) 。

无论是使用普通数组还是利用前缀和数组，对于上述两种操作，均有一种的时间复杂度为 O(n)O(n)O(n) 。而树状数组通过维护一个与 numsnumsnums 等大的，在逻辑上为树状结构 (一棵或多棵多叉树) 的数组 tree[]tree[]tree[] ，使得两种操作的时间复杂度均为 O(logn)O(logn)O(logn) 。

树状数组是一种极具巧思，代码实现极轻巧却不失高效的数据结构 。我们马上会看到树状数组如何借助 二进制形式的 numsnumsnums 的下标值 ，将 numsnumsnums 划分为多个子区间，这些子区间构成逻辑树形结构，利用树的特点使得两种基本操作都复杂度都是 O(logn)O(logn)O(logn) 。

区间划分

我们已经知道，使用普通数组或利用前缀和数组实现 PU / RQ 操作时，各自均有一种操作需要遍历 一段连续的区间 。在 numsnumsnums 上的「连续」操作的时间复杂度为 O(n)O(n)O(n) 。为了提高操作效率，我们必须 减少操作的次数 。首先考虑求长度为 kkk 的区间的区间和操作，我们会想，如果不是连续地相加 kkk 次，而是通过某种预先处理的手段，将大小为 kkk 的区间 划分为多个子区间 ，子区间个数显著地少于 kkk ，每一个子区间的区间和都被高效地实时维护，那么求区间和时，就只需要执行远少于 kkk 次的加运算 (子区间的区间和相加) 。

例如下图，当我们求 numsnumsnums 区间 [3,8][3, 8][3,8] 的区间和时，如果我们能通过某种方式，找到子区间 [3,4][3,4][3,4] 、 子区间 [5,7][5,7][5,7] 、子区间 [8,8][8,8][8,8] ，且这些子区间的区间和总是能够被实时维护，那么只需将这三个子区间的和相加即可，原先需要将 6 个数相加，现在只需要将 3 个数相加，这样就减少了操作的次数。

那么树状数组是这一想法的实现吗？答案是：不完全是。 树状数组确实将 numsnumsnums 划分成了多个区间，但并不是对任意区间 [l,r][l,r][l,r] 划分连续子区间，而是通过 「前缀区间和」作差 的方式来得到指定区间的「区间和」，前缀区间才是通过若干个连续子区间组成的。

我们提前指出，树状数组这一数据结构，对输入数组 numsnumsnums 划分为多个子区间，使得对任意的 [0,k][0,k][0,k] 前缀区间，都可以 由划分结果中的若干个连续的子区间构成 ，这些子区间的区间和相加即可得到「前缀区间和」。对于任意区间 [l,r][l,r][l,r] ，将右界前缀区间 [0,r][0, r][0,r] 的区间和减去左界前一位的前缀区间 [0,l−1][0,l-1][0,l−1] 的区间和，即为 [l,r][l,r][l,r] 区间和。

区间划分的思考也许会让你想到利用「倍增思想」的快速幂算法 leetcode #50-pow(x, n) ，该算法不是通过「连续」地将 xxx 相乘 nnn 次，而总是借助已经算出的结果来快速得到新的更大范围的中间结果，这个中间结果又能用于之后求更大范围的结果的计算中。实际上除了倍增思想，动态规划、记忆化搜索都体现了这种 「利用已求出的结果完成下一步计算」 的思想。总之我们需要一种类似倍增方法的能够跳跃式划分区间 (下标) 的方法。

现在，我们重新将 树状数组解决区间和问题 的 灵感来源 描述如下:

  • numsnumsnums 上任意 [l,r][l, r][l,r] 的区间和，将通过 [0,r][0,r][0,r] 和 [0,l−1][0, l-1][0,l−1] 的「前缀区间和」作差得到。
  • 前缀区间由若干个相邻的子区间构成，这些子区间的区间和相加得到前缀区间的区间和。
  • 也许能够通过某种类似倍增方式划分 numsnumsnums 的下标 (从倍增法得到的灵感)。

很抽象，尤其是最后一句，现在无法得知如何处理下标来划分输入区间，不过没关系，我们马上对大小为 8 (下标范围为 [0,7][0,7][0,7])的 numsnumsnums 实践上述描述。 首先明确为了实现「更快地求区间和」的需求：

  1. 将指定长度的 numsnumsnums 划分为若干区间。对 numsnumsnums 下标的划分动作应当是一种 可循环的操作 。
  2. numsnumsnums 上的任意区间 [l,r][l, r][l,r] 的区间和可由 [0,r][0,r][0,r] 区间和与 [0,l−1][0,l-1][0,l−1] 区间和作差得到，这是「前缀和」思想。这要求 [0,l−1],[0,r][0, l-1], [0,r][0,l−1],[0,r] 均由 numsnumsnums 划分结果中的 若干相邻区间 所构成。从这里可以看出， 对 numsnumsnums 的划分，不只是对 [0,n−1][0,n-1][0,n−1] 的划分，而是对 [0,k](k∈[0,n−1])[0,k](k∈[0,n-1])[0,k](k∈[0,n−1]) 的划分。划分结果必须满足能由若干连续区间构成任意的 [0,k](k∈[0,n−1])[0,k](k∈[0,n-1])[0,k](k∈[0,n−1]) 区间 (这句话是理解树状数组的关键，请读者在阅读后文时反复体会)。
  3. 子区间的区间和被 实时维护 。需要用一个 tree[]tree[]tree[] 数组保存所有子区间和 (我们马上会知道为什么命名为 treetreetree)。
  4. 求给定区间的区间和，即求两次前缀区间和再作差。求前缀区间和需要通过 某种规则 一边寻找其子区间 iii (连续的) ，一边将 tree[i]tree[i]tree[i] (子区间 iii 的区间和) 累计到结果中。
  5. 更新 nums[i]nums[i]nums[i] 时，通过 某种规则 更新包含该值的所有子区间的区间和。

「灵感来源」描述中提到了「类似倍增方式」，我们不难想到可以从下标的二进制表示着手。以划分区间 [0,14][0,14][0,14] 为例，先写出 14 的二进制表示 14=(1110)214=(1110)_214=(1110)2​ 。我们要求划分动作是可循环的操作，且对于任意长度的 numsnumsnums，都能通过同样的方式完成划分。划分区间就是要确定区间的左右界。很直接地，区间 [0,14][0,14][0,14] 的 最右子区间的右界 为 14 ， 最左子区间的左界 是 0 。我们将最右子区间作为当前区间，从当前子区间右界下标 14 开始考虑。

一个容易想到的方法如下：

【区间划分方法】：从最右子区间右界 kkk 开始，将 kkk 的最低位 1 换成 0 ，新的 kkk 作为当前区间的左界下标，减 1 即为其 左邻子区间的右界下标 。重复该操作直到当前区间右界下标为 0 (二进制数所有位都没有 1) 。

关于「树状数组」，总结本文内容如下。

  1. 基本的树状数组 (PURQ BIT) 以 O(logn)O(logn)O(logn) 时间复杂度解决长度为 nnn 的序列的 单点修改 及 区间查询 问题。
  2. 我们从区间查询出发，思考如何利用类似倍增思想的做法来划分子区间，从而提高区间查询的效率。
  3. 通过对 numsnumsnums 下标二进制形式的观察，找到了一种将输入序列划分为 nnn 个子区间的方式。链接子区间及包含它的更大一点的子区间后，这些子区间构成一棵或多棵逻辑上的多叉的 「二元索引树」 。
  4. 单点修改及区间查询的时间复杂度都与 (n)2(n)_2(n)2​ 的位数相关，简单分析后可知它们都是时间为 O(logn)O(logn)O(logn) 的操作。
  5. 在 PURQ BIT 的基础上，引入 差分数组 ，实现了 RUPQ BIT。
  6. 在 RUPQ BIT 的基础上，根据 算式推导 ，引入 辅助树状数组 ，实现了 RURQ BIT。
  7. 当区间问题只关心元素之间的大小关系而不关心元素值时，可以先将输入序列 离散化 ，有助于提高求解效率或帮助解决一些特定问题。离散化实现包括 松离散 和 紧离散 。
  8. 「求指定区间 [l,r][l,r][l,r] 内大小在指定取值范围 [lower,upper][lower, upper][lower,upper] 内的元素数」是一类特定的可用树状数组巧妙解决的问题。

普通数组+前缀和数组 O(n)O(n)O(n) O(1)O(1)O(1) O(n)O(n)O(n) O(1)O(1)O(1)

值得注意的是，RURQ BIT 对大小为1的区间执行区间修改和区间查询实际上就是单点修改和单点查询，因此 RURQ BIT 以 O(logn)O(logn)O(logn) 复杂度同时实现了 单点修改 / 单点查询 / 区间修改(增量式) / 区间查询(区间求和) 操作。
