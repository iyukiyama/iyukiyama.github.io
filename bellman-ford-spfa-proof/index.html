Bellman-Ford

算法描述

Bellman-Ford Algorithm(贝尔曼-福特算法): 与 Dijkstra 算法的相同点是对边 (或者说对顶点) 不断执行松弛操作 ，以逐渐得到所有顶点到源点的最短距离。Dijkstra每次循环完成 一个 顶点最短路径的确定，而 BF 算法则对图的 所有边 (∣E∣|E|∣E∣ 条边) ，简单地进行 ∣V∣−1|V|-1∣V∣−1 次 全量松弛操作 ，第 iii 次「全量松弛」使得位于第 i+1i+1i+1 层的顶点的距离被确定 (详见后文)，因而 BF 算法也是「贪心」思想的应用。又因为任意顶点最多居于第 ∣V∣|V|∣V∣ 层 (以源点 sss 为第 1 层)，因此算法结束时，保证 (无负圈图) 所有顶点距离最短。


算法过程

  1. 置源点 sss 的距离为 0。

  2. 外层循环执行 ∣V∣−1|V|-1∣V∣−1 次，每一次都「松弛所有边」。

  3. 进入外层循环后按顶点顺序依次对所有顶点 vvv 考察其所有邻接顶点 www 是否有 dv+∣(v,w)∣<dwdv + |(v, w)| < dwdv+∣(v,w)∣<dw，若有，松弛之，即令 dw=dv+∣(v,w)∣dw = dv + |(v, w)|dw=dv+∣(v,w)∣。

  4. 在一次「松弛所有边」的操作中，若没有任何边被松弛，则退出最外层循环，表明所有可能的松弛已完成 (负圈图除外)。

  5. 检查图是否有负圈。做法是再对所有边执行松弛操作，若有边可被松弛，则有负圈，结束程序，否则正常结束，所有顶点最短路径被求出。


正确性证明(说明)

  1. 已知若一个顶点 vvv 的所有入边若完成了所有可能的松弛 (一条入边可被多次松弛)，则在 最后一次松弛 后，必有 dv=δvdv=δvdv=δv 。 (dvdvdv 表示由算法得到的 vvv 的距离， δvδvδv 表示实际的 vvv 的最短距离)。

  2. 易知，第 iii 次全量松弛，第 iii 层顶点的出边 必被松弛 。例如第 1 次全量松弛，sss (第 1 层顶点) 的出边必被松弛。第 2 次全量松弛，由于有了第 1 次全量松弛的结果，sss 的邻接顶点 (第 2 层顶点) 的出边必被松弛。

  3. p(s,v)p(s, v)p(s,v) 路径长 (边数) 最长不会超过 ∣V∣−1|V|-1∣V∣−1 ，当 sss 到 vvv 为链状时最长 ∣V∣−1|V|-1∣V∣−1 。当 vvv 的入边是第 1 层入边时，将在第 1 次全量松弛时被松弛，若是第 2 层入边，则会在第 2 次全量松弛时被松弛 (这条入边的发出顶点在第 1 次全量松弛时已被松弛)，以此类推，第 iii 层入边会在第 iii 次全量松弛时被松弛。所有顶点的入边组成了该图的所有边，任意一边一定是某一层次的入边 (可以同时属于多个层次) 。某一层顶点距离的更新，一定来自其入边的松弛。

  4. 由此，vvv 的入边能否被全部松弛只取决于最深的入边能否被松弛 (当一条入边属于多个层次时，取其最深层次)，也即取决于 sss 到 vvv 的最长路径 max⁡{∣p(s,v)∣,v∈V}\max\{|p(s, v)|, v ∈ V\}max{∣p(s,v)∣,v∈V}的长度。如前述， max⁡{∣p(s,v)∣,v∈V}≤∣V∣−1\max\{|p(s, v)|, v ∈ V\} ≤ |V|-1max{∣p(s,v)∣,v∈V}≤∣V∣−1 ，故至多经过 ∣V∣−1|V| - 1∣V∣−1 次全量松弛，图的所有入边必定都松弛过且完成了所有可能的松弛 (某条入边属于多个层次时，可能经过多次松弛) 。如果把图看成以 sss 为根的树，也可以说为了求得所有顶点的最短路径，所需的全量松弛的次数取决于 树的高度 。对某一顶点，可以说该顶点至多经过其最大深度减 1 次全量松弛后取得最短路径。

  5. 上述过程对任意顶点均成立，故 BF 算法正确性得证。

此证明也可以看作如下「动态规划」过程。且为较简单的单串单依赖动态规划 (单依赖指当前元素的更新只依赖前一元素)。

  1. 定义: dp[j]dp[j]dp[j] 表示源点 sss 到顶点 jjj 的最短路径长度。

  2. 边界: dp[s]=0dp[s] = 0dp[s]=0 。

  3. 递推: dp[j]=dp[j−1]+∣(j−1,j)∣dp[j] = dp[j - 1] + |(j - 1, j)|dp[j]=dp[j−1]+∣(j−1,j)∣ 。j−1j - 1j−1 表示 s−js-js−j 最短路径上的前一个顶点。

根据前述，j−1j - 1j−1 顶点最大层深必比 jjj 小 1，故 dp[j−1]dp[j - 1]dp[j−1] 一定在求 dp[j]dp[j]dp[j] 之前就已求出，递推式成立。

负边图 & 负圈图

  • 负边图。由于该算法在 ∣V∣−1|V|-1∣V∣−1 次对所有边的松弛操作中会穷尽所有边被松弛的可能，类似以拓扑排序方式针对 DAG 图的 Dijkstra 算法 (通过入度为 0 保证穷尽所有松弛的可能)，所以也适用于有负边的图。

  • 负圈图。当图存在负圈时，sss 到圈上任意顶点的距离都可以通过不断绕圈趋于无限小。因此若不能保证输入的图无负圈，可以在 ∣V∣−1|V| - 1∣V∣−1 次全量松弛后再执行一次全量松弛，若仍有边可被松弛，说明存在负圈。


提前结束优化

当某一次全量松弛过程中没有边被松弛，说明所有可能的松弛已被穷尽，可提前结束程序。


最坏情形

假定每次「全量松弛」均以 v,c,b,a,sv, c, b, a, sv,c,b,a,s 的顺序松弛，程序开始时置源点 sss 的距离为 0。

  1. 第 1 次循环只有 aaa 的距离被松弛为 1。

SPFA

算法描述

SPFA算法(最短路径快速算法)：SPFA 算法是对 BF 算法的一种改进。在BF 算法的说明中我们指出，第 iii 次「全量松弛」操作，只有第 i+1i+1i+1 层的顶点距离会被更新至最短，这明显具有 BFS 特点，因此可以考虑不通过「全量松弛」来松弛第 i+1i+1i+1 层顶点，而是以 BFS 的方式，借助队列 qqq ，每松弛一层顶点，将它们入队，出队时，尝试松弛到其所有邻接顶点的距离，即可在第 iii 层顶点出队时松弛第 i+1i+1i+1 顶点并使这些第 i+1i+1i+1 顶点距离取得最小。因为顶点「按层」入队出队，层深最大为 ∣V∣|V|∣V∣ (无负圈图)，因此算法可以结束。

我们还可以这样看，一个顶点 www 的距离能够被更新，隐含着这样一个 前提 ：www 的前驱 vvv 的距离被更新过 。因为 dv+∣(v,w)∣<dwdv + |(v, w)| < dwdv+∣(v,w)∣<dw 时才会更新 dwdwdw，而 ∣(v,w)∣|(v, w)|∣(v,w)∣ 是不变的，初始时 dvdvdv 和 dwdwdw 都是无穷大，所以只有 dvdvdv 更新 (变小)，dwdwdw 才会更新 (变小)。从源点出发指向其邻接顶点，对一个连通的有向图，总能遍历所有顶点，每次考察已松弛的顶点 vvv 是否能松弛其邻接顶点 www ，www 成为已松弛的顶点后再考察是否能松弛 www 的邻接顶点，重复此操作直到「当前已松弛顶点均无法再松弛任何顶点」为止。设置一个队列 qqq，程序开始时置源点s的距离为 0，sss 入队。while 对 qqq 判空，不空时队首顶点 vvv 出队，松弛其边 (即更新 vvv 的邻接顶点 www 的距离)，根据上述分析，如果 dwdwdw 被更新，那它的邻接顶点将 有机会被更新 ，所以将 www 入队，等待出队时尝试松弛 www 的边。需要注意的是，www 入队前需要检查当前队列中是否已有 www ，若有则无需入队。该检查使得时间复杂度为 O(∣V∣∣E∣)O(|V||E|)O(∣V∣∣E∣) ，否则这一复杂度将无法得到保证。重复上述过程，当 qqq 为空时 (无负圈图) 代表所有被更新过距离的顶点，都 无法再触发其邻接顶点距离的更新 。也可以说对任意一个顶点 vvv 来说，sss 到 vvv 的所有路径带来的所有可能的对 dvdvdv 的更新已被穷尽。程序结束时得到所有顶点到源点的最短路径。


算法过程

  1. 初始时所有顶点距离为 InfinityInfinityInfinity。设置一个队列 qqq，置源点 sss 的距离为 0，sss 入队。

  2. 设置一个用于支持负圈检测的记录所有顶点入队次数的哈希表。

  3. 以 while 循环对 qqq 判空，若 qqq 不空，队首顶点 vvv 出队。

  4. 遍历 vvv 的邻接顶点 www ，若满足松弛条件，则更新 www 的距离，置 www 的前驱为 vvv ，若当前队列无 www 则将 www 入队。然后检查 www 的入队次数是否大于 ∣V∣−1|V| - 1∣V∣−1，若大于则表示存在负圈，结束程序，否则将 www 入队次数加 1。


正确性证明(说明)

SPFA 算法与 BF 算法的核心内容都在于 穷尽所有路径带来的所有可能的松弛 。BF 算法通过 ∣V∣−1|V|-1∣V∣−1 次全量松弛达到 (详见BF算法正确性证明)，第 iii 次全量松弛中，有效松弛仅作用于第 i+1i+1i+1 层顶点，其他层深顶点已不能够被松弛却还是会被尝试，这就产生了冗余操作。SPFA 算法利用顶点的距离能够被松弛的隐含前提，通过队列来减少松弛顶点距离的次数。第 iii 层顶点出队时发生的松弛，效果上相当于 BF 算法外层循环第 iii 次对所有边的全量松弛。在连通且无负圈的情况下，按层推进一定能够执行最大层深第 ∣V∣|V|∣V∣ 层，因此该算法是正确的。

按层分析很容易得到 SPFA 的复杂度。顶点一层一层地入队出队，一张图最多有 ∣V∣|V|∣V∣ 层 (以 sss 为第 1 层)，所以 按层计 ，任何顶点最多只能入队 ∣V∣−1|V| - 1∣V∣−1 次。第 1 层顶点个数为 1，其余每层顶点数不会超过 ∣V∣−1|V| - 1∣V∣−1 （第8行所保证）。再次强调，虽然一个顶点可能会通过不同的路径重复属于某一层，例如 s>a>c，s>b>cs > a > c，s > b > cs>a>c，s>b>c ，但有了第 8 行的检查，使得一个顶点最多只能在一层顶点里出现一次。考虑每层顶点个数小于 ∣V∣|V|∣V∣ ，每层顶点的松弛次数少于 ∣E∣|E|∣E∣ 次，因此复杂度为 O(∣V∣∣E∣)O(|V||E|)O(∣V∣∣E∣) 。

第 8 行 if(!q.contains(w)) 是SPFA 作为改进 BF 的关键，有必要继续进一步说明为何加了这个检查优化不影响结果的正确性。假设从 sss 经过长度相同的不同路径到达若干个不同顶点，这些顶点都指向 vvv ，每条路径带来的松弛都能执行到 (BFS 所保证)，只是除了第一次之外不把 vvv 放入队列。将 vvv 放入队列的目的是在之后使其邻接顶点 www 有被松弛的机会。对于 www ，来自 vvv 的松弛机会只需一次即可，所以不每次都将 vvv 放入队列中。

SPFA 每出队第 iii 层顶点，使得在最短路径上第 i+1i+1i+1 层的顶点得到松弛。不厌其烦地，SPFA第 iii 层顶点出队的效果等同于 BF 第 iii 轮对边的全量松弛的效果，比 BF 的操作更有效率的地方在于，SPFA 仅仅松弛它够得着得邻边。BF 暴力地松弛所有边，但有效的只有第 i+1i+1i+1 层，其他更深或更浅的顶点无法松弛，但一律以 if(dv + |(v, w)| < dw) 询问了一次。所以说 SPFA 是 BF 的一种改进。

空间复杂度：用于支持负圈检测的哈希表的空间，O(∣V∣)O(|V|)O(∣V∣) 。

对时间复杂度的分析可以看出，稀疏图中顶点的 p(s,v)p(s, v)p(s,v) 路径平均条数很少，SPFA 实际运行速度会很快。但在稠密图下会达到 O(∣V∣∣E∣)O(|V||E|)O(∣V∣∣E∣) 。


Bellman-Ford-Moore 和 SPFA

SPFA 实际上应当称作 Bellman-Ford-Moore 算法。根据Wiki词条 Bellman-Ford algorithm 的介绍，「对所有的边，简单地松弛 ∣V∣−1|V| - 1∣V∣−1 轮」的朴素 BF 算法在相近的几年里被三个人分别独立发明。只是不知道什么原因算法名称后来定型成了 Bellman-Ford。

又过了一年，1959年的时候Edward F. Moore提出了BF算法的一个改进，即前文的伪代码 (SPFA / Bellman-Ford-Moore) 。

A variation of the Bellman-Ford algorithm known as Shortest Path Faster Algorithm, first described by Moore (1959), reduces the number of relaxation steps that need to be performed within each iteration of the algorithm.

1994年，西南交通大学的段凡丁在该年4月的《西南交通大学学报》里发表了题为《关于最短路径的SPFA快速算法》的论文，重新提出了Moore的改进 ，并且给了个比较通俗的名字 Shortest Path Fast Algorithm。段老师显然没看过 Moore 当初的论文，否则不会给出一个错误的复杂度估计（给出的复杂度是 O(k∣E∣)O(k|E|)O(k∣E∣) ）。有意思的是，现在用 Google 搜 SPFA，即便在英文论坛，许多人对这个改进也称之为 SPFA，而非 BFM，真可谓是「喧宾夺主」了。
