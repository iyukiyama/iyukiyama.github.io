3-无重复字符的最长子串

3-无重复字符的最长子串 (中等)


解法一：暴力搜索 (超时)

算法描述

用两个循环，遍历所有子串可能，维护一个 maxmaxmax 实时更新。每得到一个子串，调用boolean noRepeat()方法来判断该子串是否有重复字符，如无，则更新 maxmaxmax。

时空复杂度

时间复杂度：两个 forforfor 加上每次 noRpeatnoRpeatnoRpeat 调用，整体为 O(n3)O(n^3)O(n3) 复杂度。

空间复杂度：在 noRpeatnoRpeatnoRpeat 中维护了一个 HashSetHashSetHashSet，最大不超过字符串字符总数 nnn 或字符集大小 mmm ，为 O(max(n,m))O(max(n,m))O(max(n,m)) 。

解法二：HashMap + 跳跃式滑动窗口

算法描述

在解法一思路的基础上，遇到一个字符，若不重复，将字符本身连同下标都保存到 HashMapHashMapHashMap 中， 重复时，就可以以此字符为 keykeykey 在当前 mapmapmap 中找到其 valuevaluevalue ，即该字符第一次出现在 sss 中的下标，用这个下标直接更新窗口左界，实现 跳跃式窗口滑动 。从而时间复杂度会用 setsetset 的解法的 O(2n)O(2n)O(2n) 优化到为 O(n)O(n)O(n) 。


时空复杂度

时间复杂度：遍历一遍 sss 的字符，O(n)O(n)O(n) 。

空间复杂度：为 O(m)O(m)O(m) ，mmm 是字符集大小。

解法三：哈希数组 + 跳跃式滑动窗口

算法描述

根据题目提示，输入字符串只包含英文字母，数字，符号和空格。是 ASCIIASCIIASCII 码能够覆盖的范围，最大不重复子串最多只有 128 个，因此可以用长度 128 的数组 hashhashhash 来保存字符在 sss 中的下标，减少哈希函数计算的开销。从起点的 0 宽窗开始，窗的右界（右指针）向右逐位扩展，考察进入的字符是否与窗内字符重复，若重复，则将窗左界（左指针）跳到重复字符的前一位，然后以当前窗大小更新 maxmaxmax 。 需要注意的是，在跳跃时，重复字符 (下标 iii ) 可能出现在左界 (下标 lll ) 之前，因此跳跃目标点应取 max(l,i+1)max(l, i + 1)max(l,i+1) 。例如 s = "abcdebgfa" ，遇到第二个 bbb 时，发现已存在 bbb ，且当前记录的 bbb （第一个 bbb ）的位置大于当前窗左界，因此新的左界要移动到第一个 bbb 的前一位，得到新窗 "cdeb" 。窗继续扩展直到遇到第二个 aaa ，当前记录的 aaa （第一个 aaa ）的位置小于当前窗左界（ ccc 处），因此保持窗左界不动。

在「代码」#1 处，hashhashhash 数组初始时元素值均为 0，因此在考察当前字符时，考察其在 hashhashhash 数组对应位置是否已经记录了其在 sss 中的下标，若为初始的 0，说明当前字符为首次出现，否则需调整窗左界。这里存在一个问题，即 sss 中的首个字符下标为 0，因此该字符第二次出现时也满足 hash[idx] == 0 。为了规避这种情形，可以在一开始将 hashhashhash 的每一个值置为 -1，但需要一次循环，这里考虑将 hashhashhash 的元素值表示为在 sss 中的下标 +1（「代码」中的 #2），这样一来，只要 hash[idx] == 0 ，就一定表示该字符是首次出现。


时空复杂度

时间复杂度：与哈希表法相同，O(n)O(n)O(n) ，但由于节省了哈希函数的计算开销，优于哈希表法。

空间复杂度：为 O(m)O(m)O(m) ，mmm 是字符集大小。
